# Beta User Guide

## Convite

O beta fechado é uma validação controlada. O objetivo é aprender com poucos usuários, proteger dados sensíveis e corrigir atritos antes de qualquer abertura maior.

## Antes de entrar

O usuário beta deve saber:

- O produto ainda está em beta.
- Algumas integrações reais podem estar desativadas.
- IA real e e-mail real só serão ligados se aprovados.
- Dados sensíveis não devem ser enviados em feedback, prints ou suporte.
- Atalaia só deve ser usado quando consentimento, prévia e revogação estiverem validados.

## Primeira sessão sugerida

1. Criar conta e entrar.
2. Completar perfil essencial.
3. Completar Mapa da Vida.
4. Criar Chamado provisório ou hipótese em discernimento.
5. Criar primeiro alvo.
6. Gerar projeto/tarefa.
7. Agendar uma ação.
8. Executar foco curto ou usar Desbloqueador.
9. Registrar feedback se algo travar.

## Uso seguro no beta

- Use dados fictícios nos primeiros testes de preview.
- Não envie senhas, tokens, dados médicos, financeiros, familiares ou conteúdo íntimo por feedback.
- Não envie prints com Metacognição, Chamado, calendário ou dados pessoais.
- Em dúvida de privacidade, pause o uso daquele fluxo e avise suporte.

## Canal de feedback

O app prepara feedback in-app em modo beta/local. O envio externo depende de formulário/canal aprovado pelo fundador.

Campos esperados:

- Módulo.
- O que funcionou.
- O que confundiu.
- O que travou.
- Nota de clareza.
- Nota de utilidade.
- Nota de peso/fricção.
- Comentário curto opcional.
