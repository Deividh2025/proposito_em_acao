# Mobile UX Guide

## Regra central

Abrir, tocar, registrar e fechar.

O desktop continua sendo o lugar para planejamento, edicao profunda, revisao semanal, calendario completo e decisoes complexas. O mobile e para registrar movimento no momento.

## Padroes

- Uma acao principal por tela.
- Poucos campos obrigatorios.
- Botoes grandes e labels curtos.
- Feedback imediato apos salvar ou marcar.
- Links para fluxo completo como acao secundaria.
- Sem dashboard completo no primeiro viewport.
- Sem historico longo, graficos densos ou configuracao antes da marcacao.

## Baixa energia

Quando energia for baixa, priorizar:

- foco de 5 minutos;
- versao minima;
- Desbloqueador;
- descanso legitimo;
- retomada sem culpa.

Nao usar linguagem moralizante, vermelho agressivo, ranking ou streak punitivo.

## Fluxos

- Captura: escrever frase, salvar, fechar.
- Habitos: marcar minimo, ideal, retomada ou pausa consciente.
- Placar: marcar feito, parcial ou retomado.
- Foco: escolher 5/15 minutos, iniciar, capturar distracao opcional, concluir.
- Desbloqueador: tarefa, trava, energia/tempo simples, primeiro passo.
- Metacognicao: estado, intensidade, pensamento, fato x interpretacao, proxima acao.
- Energia: baixa/media/alta, observacao opcional, sugestao curta.

## Criterios negativos

Nao aprovar se o mobile virar copia do desktop, se exigir muitos passos para marcacao simples, se cachear dados sensiveis ou se prometer sync offline que nao existe.
